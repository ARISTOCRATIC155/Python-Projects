# 🚀 Git & GitHub Quick Start Guide

This README helps you remember the essential Git commands for managing and updating your projects.

---

# 🧩 1. First Time Setup (New Project)

Use this when starting a project for the first time:

```bash
git init
git add .
git commit -m "first commit"
git branch -M main
git remote add origin <your-repository-url>
git push -u origin main
```

---

# 🔄 2. Daily Updates (After Changes)

Use this when you add or edit files:

```bash
git add .
git commit -m "your message here"
git push
```

Example:

```bash
git commit -m "added new Python projects"
```

---

# 📦 3. Check Project Status

See what files changed:

```bash
git status
```

---

# 📥 4. Get a Project from GitHub

If you're starting on a new device:

```bash
git clone <repository-url>
```

---

# 🔗 5. Connect Local Project to GitHub

If repo is already created on GitHub:

```bash
git remote add origin <repository-url>
```

---

# 🌿 6. Branch Rename (Recommended)

Ensure main branch is called "main":

```bash
git branch -M main
```

---

# ⚡ Quick Workflow Summary

Think of Git like this:

```
EDIT → ADD → COMMIT → PUSH
```

---

# 💡 Important Notes

* Commit messages are for humans, not required for Git to work
* Always use clear messages for future reference
* `git add .` stages all changes
* `git push` uploads to GitHub

---

# 🧠 Example Full Update Cycle

```bash
git add .
git commit -m "updated project"
git push
```
